\section{What the system does}

An SME drops microbiology SOPs into S3. An agent pipeline, grounded in a hybrid retrieval substrate (BM25 + dense), reads them and emits two artifacts: \textbf{(A)} Jira-ready Epic/Story tree and \textbf{(B)} per-culture YAML configuration profiles (urine, blood, target-pathogen). The Cyto Jira backlog is a \emph{teaching corpus} (few-shot exemplars), not a registry. The agent stops at Story --- Tasks are dev-authored.

\textbf{What each box does and emits:}

\begin{enumerate}
  \item \textbf{Epic Extractor} --- identifies capability domains. \emph{Out:} \texttt{\{id, title, discipline, themes[], cyto\_epic\_analog?\}}.
  \item \textbf{Story Extractor} --- per-SOP, multi-shape. Each Story declares its \texttt{shape} for downstream routing.
  \item \textbf{Validator gate \#1} --- shape verification + shape-specific sub-rubric. Two revisions then SME-escalate.
  \item \textbf{Cross-SOP Synthesis} --- runs after gate \#1 clears the entire batch. Behavioral-similarity clustering over (title + AC + source excerpt); a cluster qualifies only if its members come from $\geq 2$ \emph{distinct} SOPs. Lifts a capability Story per cluster, with \texttt{cross\_links[]} back to the contributing concretes.
  \item \textbf{Validator gate \#2} --- same agent as 3a; capability rubric only. Synthesized stories are a \emph{new} artifact, not a derivative, so they re-pass the bar.
  \item \textbf{Dependency Resolver} --- story$\leftrightarrow$story dependencies, stage-split sibling links, capability$\leftrightarrow$child links. Finalizes the artifact graph for Jira push.
\end{enumerate}

\textbf{Sub-rubrics by shape (cheat sheet):}
\begin{itemize}
  \item \textbf{Capability:} MUST/SHALL; testable AC; no ambiguous quantifiers; configurability boundaries; SOP citation; complexity S/M/L; edge cases enumerated.
  \item \textbf{Stage-split:} stage explicit in title; entry/exit observable; siblings enumerated and cross-linked; transition contract; boundary cases.
  \item \textbf{Configuration-instance:} typed concrete values; target table/profile named; SOP excerpt verbatim; scope (culture/lab) stated. \emph{MUST/SHALL not required.}
  \item \textbf{Cleanup:} target artifact named (id, screen path, file path); before/after observable; regression risk acknowledged; DoD includes smoke test. \emph{MUST/SHALL not required.}
\end{itemize}

\section{What ships + cross-cutting}
% =====================================================================

\begin{itemize}
  \item \textbf{Output A --- Jira artifacts.} Epics (with optional \texttt{cyto\_epic\_analog}), per-SOP concrete Stories in 4 shapes, synthesized capability Stories cross-linked to children, dependency edges between them.
  \item \textbf{Output B --- Per-culture YAML profile.} One per culture in scope; values typed (units / enums / nullable rules); SOP excerpt cited inline; \texttt{related\_story\_ac} links to Story for traceability.
\end{itemize}

\textbf{Example YAML excerpt} (\texttt{cultures/urine.yaml}):

\noindent\texttt{\scriptsize centrifugation:\\
\hspace*{1em}speed:\ \ \ \{value: 1500, units: g\}\ \ \ \# cite: SOP-MICRO-014 \S 3.2 L41--43\\
\hspace*{1em}duration: \{value: 5,\ \ \ units: min\}\ \# cite: SOP-MICRO-014 \S 3.2 L44\\
\hspace*{1em}temp:\ \ \ \ \{value: 4,\ \ \ units: degC\} \# cite: SOP-MICRO-014 \S 3.2 L45\\
related\_story\_ac: STORY-1042\#AC-2}

\textbf{Cross-cutting:} KMS encryption (rest + transit); IAM least-privilege + VPC endpoints (Bedrock calls stay in-VPC); DLQ for parse/OCR/embed failures; CloudWatch metrics + alarms (e.g.\ OCR fail-rate $> 5\%$); CloudTrail audit log; idempotent ingestion via content-hash dedupe.

\textbf{Generalization:} when Histology arrives, the same agents and the same retrieval substrate run --- only the exemplar corpus and config priors get extended. No model retraining.
